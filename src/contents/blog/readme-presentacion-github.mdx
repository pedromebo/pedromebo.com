> ## Introducción

En el post de hoy vamos a ver cómo crear tu propio perfil de presentación README en GitHub.

Si aún no lo sabes, GitHub es un sistema de gestión de proyectos y control de versiones de código. El principal uso es trabajar en colaboración con otras personas, planificar proyectos y realizar un seguimiento del trabajo.

Cada usuario tiene un perfil donde almacenar sus repositorios privados, tener acceso a los públicos y a los que estén compartidos con él. Además, el usuario tiene la oportunidad de incluir un perfil para mostrar al resto de usuarios, hablar un poco de él, de sus proyectos o mostrar estadísticas de su perfil. ¡Eso es lo que vamos a hacer hoy!

> ## Creación del repositorio

Para comenzar, lo primero que debemos crear es un repositorio que tenga el mismo nombre que nuestro usuario. Tendremos que hacerlo público y seleccionar la opción que nos permite crear un README al inicio.

![Readme-Github](https://i.gyazo.com/7916574cac1768c8a9bdb1dbd6110591.png)

> ## Configuración del README

Una vez el repositorio está creado solo nos queda configurar el README y ponerlo como más nos guste. En mi caso voy a utilizar referencias que he encontrado en una lista "Awesome-Readme". Si no sabes qué son las listas awesome, en el post de la semana pasada hablamos sobre ellas. [Click aquí para verlo.](https://www.pedromebo.com/awesome-listas-de-recursos-para-programacion)

Algunos de los elementos que no pueden faltar son:

- Introducción
- ¿Qué estudias? ¿En qué trabajas?
- ¿Últimos proyectos o proyectos en activo?
- Redes sociales
- Stats
- Proyectos

Puedes decorarlo y añadir la información que quieras, recuerda que es un buen espacio para darte a conocer, ¡aprovéchalo!
